# PRD (Product Requirements Document)
> 가계부 웹앱 — 개인용 → 배포 확장 가능 구조

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | 가계부 앱 (이름 미정) |
| 목적 | 개인 수입/지출 관리 |
| 초기 타겟 | 본인 1인 사용 |
| 최종 타겟 | 일반 사용자 배포 |
| 플랫폼 | 웹 (PWA, 모바일/PC 모두 지원) |
| 배포 환경 | AWS EC2 프리티어 |

---

## 2. 사용자 스토리

### 1단계 (현재 — 1인 사용)
- 나는 오늘 쓴 돈을 빠르게 입력하고 싶다
- 나는 이번 달 얼마나 썼는지 한눈에 보고 싶다
- 나는 카테고리별로 지출을 분류해서 보고 싶다
- 나는 월별 지출 추이를 차트로 보고 싶다
- 나는 카테고리별 예산을 설정하고 초과 여부를 알고 싶다
- 나는 정기 지출(구독료, 월세 등)을 자동으로 입력하고 싶다

### 2단계 (배포 이후)
- 사용자는 회원가입/로그인을 할 수 있다
- 사용자는 여러 기기에서 데이터를 동기화할 수 있다
- 사용자는 소셜 로그인(Google 등)으로 가입할 수 있다

---

## 3. 기능 목록

### 핵심 기능 (MVP)
| 기능 | 설명 | 우선순위 |
|---|---|---|
| 거래 입력 | 수입/지출 금액, 카테고리, 날짜, 메모 입력 | P0 |
| 거래 수정/삭제 | 입력한 내역 편집 및 삭제 | P0 |
| 거래 내역 조회 | 월별 리스트, 필터, 검색 | P0 |
| 대시보드 | 이번 달 요약, 예산 진행률, 최근 내역 | P0 |
| 카테고리 관리 | 기본 카테고리 제공, 추가/수정/삭제 | P0 |
| 월별 통계 | 수입/지출 추이 차트, 카테고리별 비율 | P1 |
| 예산 설정 | 월별/카테고리별 예산 설정 및 초과 알림 | P1 |
| 계좌/카드 관리 | 복수 계좌 등록 및 거래에 연결 | P1 |
| 정기 지출 | 반복 거래 자동 생성 | P2 |
| CSV 내보내기 | 거래 데이터 엑셀 내보내기 | P2 |

### 인증 기능
| 기능 | 1단계 | 2단계 |
|---|---|---|
| 로그인 | 하드코딩 계정 1개 | 회원가입/로그인 |
| 인증 방식 | JWT | JWT + 소셜 로그인 |
| 멀티유저 | ❌ | ✅ |

## 4. 기본 카테고리

### 지출
식비, 카페/간식, 교통, 쇼핑, 문화/여가, 의료/건강, 통신, 구독, 월세/관리비, 기타

### 수입
월급, 부수입, 용돈, 이자, 환급, 기타

---

## 5. 비기능 요구사항

- 모바일 퍼스트 반응형 UI
- PWA 지원 (홈 화면 설치 가능)
- 페이지 로딩 2초 이내
- EC2 프리티어(t2.micro)에서 정상 동작
- 추후 PostgreSQL 마이그레이션 가능한 DB 설계

- 은행 API 자동 연동

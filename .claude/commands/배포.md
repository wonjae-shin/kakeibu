# 배포 파이프라인

아래 단계를 순서대로 실행하고, 각 단계 결과를 추적하라. 단계별로 성공/실패를 명확히 보고하라.

## 단계 0 — 사전 점검

```bash
git status
git branch --show-current
```

- 현재 브랜치가 `main`인지 확인
- 미커밋 변경사항이 있으면 목록을 출력 (배포 흐름은 계속 진행)

## 단계 1 — 서버 테스트

```bash
cd server && npm test 2>&1
```

- **실패 시**: 테스트 실패 내용을 출력하고 이후 모든 단계를 중단. 결과 요약에 FAIL 표시.
- **성공 시**: 다음 단계로 진행.

## 단계 2 — 클라이언트 테스트

```bash
cd client && npm test 2>&1
```

- **실패 시**: 테스트 실패 내용을 출력하고 이후 모든 단계를 중단. 결과 요약에 FAIL 표시.
- **성공 시**: 다음 단계로 진행.

## 단계 3 — 클라이언트 빌드 검증

```bash
cd client && npm run build 2>&1
```

- **실패 시**: 빌드 오류를 출력하고 이후 모든 단계를 중단. 결과 요약에 FAIL 표시.
- **성공 시**: 다음 단계로 진행.

## 단계 4 — Git 커밋 & 푸시

미커밋 변경사항이 있는 경우에만 실행:

```bash
git add -A
git status
```

변경된 파일 목록을 보고 Conventional Commits 규칙에 따라 적절한 커밋 메시지를 작성하라.
- 새 기능: `feat: ...`
- 버그 수정: `fix: ...`
- 리팩토링: `refactor: ...`
- 설정/환경: `chore: ...`

```bash
git commit -m "<생성한 커밋 메시지>"
git push origin main
```

미커밋 변경사항이 없는 경우, 이 단계는 건너뜀.

## 단계 5 — 결과 요약

아래 형식으로 최종 요약을 출력하라:

```
==============================
      배포 파이프라인 결과
==============================
브랜치       : main
실행 시각    : <현재 날짜/시간>

[ 단계별 결과 ]
  서버 테스트       : ✅ PASS / ❌ FAIL
  클라이언트 테스트  : ✅ PASS / ❌ FAIL
  클라이언트 빌드   : ✅ PASS / ❌ FAIL
  Git 커밋 & 푸시  : ✅ 완료 / ⏭️ 스킵(변경 없음) / ❌ FAIL

[ 최종 상태 ]
  ✅ 배포 준비 완료 — 원격 저장소에 푸시됨
  또는
  ❌ 배포 중단 — <실패한 단계> 에서 오류 발생
==============================
```

실패한 단계가 있으면 오류 원인과 해결 힌트를 요약에 함께 포함하라.
